import java.util.Scanner;

public class Calculator {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        System.out.print("Enter operator (+,-,*,/): ");
        char operator = sc.next().charAt(0);
        
        System.out.print("Enter two numbers: ");
        int a = sc.nextInt();
        int b = sc.nextInt();
        
        switch (operator) {
            case '+':
                System.out.println("Result: " + (a + b));
                break;
            case '-':
                System.out.println("Result: " + (a - b));
                break;
            case '*':
                System.out.println("Result: " + (a * b));
                break;
            case '/':
                System.out.println("Result: " + (a / b));
                break;
            default:
                System.out.println("Invalid operator");
        }
        
        sc.close();
    }
}

import java.util.Scanner;

public class FactorialCalculator {
    
    // Function to calculate factorial
    public static int factorial(int n) {
        if (n == 0 || n == 1) {
            return 1;
        }
        
        int result = 1;
        for (int i = 2; i <= n; i++) {
            result *= i;
        }
        return result;
    }
    
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        System.out.print("Enter a number: ");
        int num = sc.nextInt();
        
        int fact = factorial(num);
        System.out.println("Factorial of " + num + " is: " + fact);
        
        sc.close();
    }
}

Example: Prime Number Checker

import java.util.Scanner;

public class PrimeChecker {
    
    // Function to check if number is prime
    public static boolean isPrime(int n) {
        if (n <= 1) {
            return false;
        }
        
        for (int i = 2; i <= Math.sqrt(n); i++) {
            if (n % i == 0) {
                return false;
            }
        }
        return true;
    }
    
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        System.out.print("Enter a number: ");
        int num = sc.nextInt();
        
        if (isPrime(num)) {
            System.out.println(num + " is a Prime number");
        } else {
            System.out.println(num + " is not a Prime number");
        }
        
        sc.close();
    }
}

Key Points:

  • Functions promote code reusability and modularity
  • return statement sends value back to the caller
  • void functions don't return any value
  • Use descriptive names and keep functions focused on one task

        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= n - i; j++) {

import java.util.Scanner;

public class ArrayMax {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        System.out.print("Enter array size: ");
        int n = sc.nextInt();
        
        int[] arr = new int[n];
        
        System.out.println("Enter " + n + " elements:");
        for (int i = 0; i < n; i++) {
            arr[i] = sc.nextInt();
        }
        
        int max = arr[0];
        for (int i = 1; i < arr.length; i++) {
            if (arr[i] > max) {
                max = arr[i];
            }
        }
        
        System.out.println("Maximum element: " + max);
        sc.close();
    }
}

Introduction to OOP: Classes & Objects + 4 Pillars of OOP

Key Concepts

OOP structures code around reusable objects. Here's how it flows:

  • Class: Blueprint/template for creating objects
  • Object: Instance of a class (e.g., `Car myCar;`)
  • Attributes: Variables that store object data (e.g., `color`, `speed`)
  • Methods: Functions that define object behavior (e.g., `drive()`, `brake()`)

The 4 Pillars of OOP

1. ðŸ”’ Encapsulation

Concept: Bundling data and methods, and hiding internal state (data security).

// Data Hiding
private String name;
    
// Controlled Access
public void setName(String n) {
    if (n.length() > 0) {
        name = n;
    }
}

Analogy: A pill capsule contains medicine, hiding its complex contents but providing safe, public access via consumption.

2. ðŸ§  Abstraction

Concept: Showing only essential features and hiding complex implementation details (simplicity).

// Abstract definition
abstract class Vehicle {
    abstract void run(); // What to do
}
    
// Hidden implementation
class Car extends Vehicle {
    void run() { 
        // Complex engine logic here... 
    }
}

Analogy: You use a TV remote to change the channel without knowing how the complex signal processing works inside the TV.

3. ðŸ§¬ Inheritance

Concept: Creating new classes based on existing ones, inheriting properties and methods (reusability).

// Parent Class
class Animal {
    void eat() { ... } 
}
    
// Child Class
class Dog extends Animal {
    // Inherits eat() method!
    void bark() { ... }
}

Analogy: Biological inheritance. A Child inherits characteristics from the Parent but also has their own unique traits.

4. ðŸŽ­ Polymorphism

Concept: A single name (method) having multiple forms or functionalities (flexibility).

// Method Overloading
int add(int a, int b) { ... }
double add(double a, double b) { ... }
    
// Method Overriding
Parent p = new Child();
p.method(); // Executes Child's version

Analogy: A 'talk' button. When pressed, a dog 'barks', a cat 'meows', but the button ('method') name remains the same.

public class Student {
    // Private attributes (Encapsulation)
    private String name;
    private int rollNumber;
    private double[] marks;
    
    // Constructor
    public Student(String name, int rollNumber, double[] marks) {
        this.name = name;
        this.rollNumber = rollNumber;
        this.marks = marks;
    }
    
    // Getter methods (Accessors)
    public String getName() {
        return name;
    }
    
    public int getRollNumber() {
        return rollNumber;
    }
    
    public double[] getMarks() {
        return marks;
    }
    
    // Setter methods (Mutators)
    public void setName(String name) {
        this.name = name;
    }
    
    public void setMarks(double[] marks) {
        this.marks = marks;
    }
    
    // Utility method
    public double average() {
        double sum = 0;
        for (double mark : marks) {
            sum += mark;
        }
        return sum / marks.length;
    }
    
    // Display method
    public void display() {
        System.out.println("Name: " + name);
        System.out.println("Roll: " + rollNumber);
        System.out.println("Average: " + average());
    }
}

Method Overloading (Compile-time Polymorphism)

Same method name with different parameters in the same class.

public class Calculator {
    // Overloaded add methods
    
    // Two integers
    public int add(int a, int b) {
        return a + b;
    }
    
    // Three integers
    public int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // Two doubles
    public double add(double a, double b) {
        return a + b;
    }
    
    // Array of integers
    public int add(int[] numbers) {
        int sum = 0;
        for (int num : numbers) {
            sum += num;
        }
        return sum;
    }
}

public class OverloadingDemo {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        System.out.println("2 + 3 = " + calc.add(2, 3));
        System.out.println("1 + 2 + 3 = " + calc.add(1, 2, 3));
        System.out.println("2.5 + 3.7 = " + calc.add(2.5, 3.7));
        
        int[] nums = {10, 20, 30};
        System.out.println("Array sum = " + calc.add(nums));
    }
}

Key Points:

  • Same method name, different parameters
  • Decided at compile time
  • Parameters differ in number, type, or order
  • Makes code more readable and flexible

Method Overriding (Runtime Polymorphism)

Child class provides specific implementation of parent's method.

// Parent class
class Animal {
    public void sound() {
        System.out.println("Animal makes a sound");
    }
    
    public void sleep() {
        System.out.println("Animal is sleeping");
    }
}

// Child class - Dog
class Dog extends Animal {
    @Override  // Annotation to indicate overriding
    public void sound() {
        System.out.println("Dog barks: Woof Woof!");
    }
    // sleep() is inherited as-is
}

// Child class - Cat
class Cat extends Animal {
    @Override
    public void sound() {
        System.out.println("Cat meows: Meow Meow!");
    }
}

public class OverridingDemo {
    public static void main(String[] args) {
        Animal myAnimal = new Animal();
        Animal myDog = new Dog();
        Animal myCat = new Cat();
        
        myAnimal.sound();  // Animal makes a sound
        myDog.sound();     // Dog barks: Woof Woof!
        myCat.sound();     // Cat meows: Meow Meow!
        
        myDog.sleep();     // Inherited from Animal
    }
}

Key Points:

  • Child class redefines parent's method
  • Same signature (name + parameters)
  • Decided at runtime based on object type
  • Use @Override annotation for clarity

Constructors & 'this' Keyword

What is a Constructor?

A constructor is a special method that initializes objects when they are created. It has the same name as the class and no return type.

Types of Constructors

  • Default Constructor: No parameters, Java provides one if you don't
  • Parameterized Constructor: Takes arguments to initialize attributes

The 'this' Keyword

The this keyword refers to the current object instance. It's used to:

  • Differentiate between instance variables and parameters
  • Call other constructors in the same class
  • Pass the current object as a parameter

        s1.display();
        
        // Using default constructor
        Student s2 = new Student();
        s2.display();
    }
}

Key Points:

  • Constructor name must match class name
  • Constructors have no return type (not even void)
  • this keyword differentiates parameters from instance variables
  • Constructors are called automatically when object is created

Debug & Demo Time

Common Errors to Watch For

1. NullPointerException
// Problem
Student s = null;
s.displayInfo(); // Error!

// Solution
Student s = new Student("John", 101, new double[]{80, 90});
s.displayInfo(); // Works!
2. ArrayIndexOutOfBoundsException
// Problem
int[] arr = new int[5];
arr[5] = 10; // Error! Index 0-4 only

// Solution
arr[4] = 10; // Correct
3. Constructor Name Mismatch
// Problem
public class Student {
    public student() { } // Wrong! Lowercase 's'
}

// Solution
public class Student {
    public Student() { } // Correct!
4. Forgetting 'this' Keyword
// Problem (ambiguous)
public Student(String name) {
    name = name; // Assigns parameter to itself!
}

// Solution
public Student(String name) {
    this.name = name; // Assigns to instance variable
}

Debugging Tips

  • Print Statements: Use System.out.println() to trace execution
  • Check Initialization: Ensure objects and arrays are properly initialized
  • Verify Indices: Array indices start at 0, not 1
  • Review Constructor: Check name matches class and parameters are used correctly
  • Test Incrementally: Test each method individually before combining
